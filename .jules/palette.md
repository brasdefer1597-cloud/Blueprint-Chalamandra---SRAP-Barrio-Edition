## 2024-05-24 - Interactive Custom Components Needs Explicit Focus Styles
**Learning:** This app heavily relies on generic `<div>` tags mapped to behave as custom buttons (e.g., `.srap-step`, `.chaos-ritual`, `.mandala-hat`). While JavaScript adds ARIA roles and keyboard interactions, default browser focus rings do not style these `<div>` tags visibly, leaving keyboard-only users blind to their active focus state.
**Action:** When creating or maintaining custom interactive elements (acting as buttons but using non-button semantic tags like `<div>`), I must always ensure programmatic or CSS injection of focus-visible utility classes (e.g. `focus-visible:ring-2 focus-visible:ring-lime-400 focus-visible:outline-none`) combined with `tabindex="0"` to bridge this visual accessibility gap without modifying custom CSS.
